# Troubleshooting

This guide helps you diagnose and resolve common issues encountered during installation, DNS configuration, SSL certification, or ERP connection setups.

## Infrastructure Troubleshooting

### 1. Ports 80/443 Blocked
Caddy requires ports `80` (for ACME HTTP-01 challenge verification) and `443` (for HTTPS traffic) to be available on the host interface.
* **Symptom:** The installation script times out, or the dashboard is unreachable in the browser.
* **Verification:** Run the following command on your server to verify if another web service (e.g., Apache or Nginx) is binding to these ports:
  ```bash
  sudo ss -tulpn | grep -E ':(80|443)'
  ```
* **Resolution:** Stop and disable the conflicting web server, or adjust your host firewall (e.g., `ufw allow 80/tcp` and `ufw allow 443/tcp`).

### 2. DNS Misconfiguration or Propagation Delays
* **Symptom:** Caddy fails to obtain an SSL certificate from Let's Encrypt, resulting in SSL handshake errors in the browser.
* **Verification:** Verify that your domain points to the server's public IP address:
  ```bash
  host jaanos.your-domain.com
  ```
* **Resolution:** Ensure the DNS A record (and AAAA if configured) points to your server's public IPv4 address. Newly registered domains can take up to 24 hours to propagate globally. If necessary, use the IP-based fallback domain `[IP-ADDRESS].sslip.io`.

### 3. Missing Docker or Compose Plugin
* **Symptom:** The script outputs command-not-found errors during setup.
* **Resolution:** Install the Docker Compose plugin using the official package repositories for your distribution (e.g., `apt install docker-compose-plugin`).

## Diagnosing ERP Connection Failures

When testing connections to Tryton or Lexware in the Settings interface, JaanOS maps technical exceptions to human-readable instructions:

* **“Connection refused” (ECONNREFUSED)**
  * **Meaning:** The target server actively rejected the connection request.
  * **Resolution:** Verify that the Tryton service is running on the target server and listening on the specified port. Tryton defaults to port `8069` or `8000`.
* **“Timeout” (TimeoutError / AbortError)**
  * **Meaning:** The request timed out after waiting for 5 seconds without a response.
  * **Resolution:** Ensure that firewalls (on either your server or the ERP host) are not blocking port access, and verify the spelling of the URL.
* **“Host not found” (ENOTFOUND / EAI_AGAIN)**
  * **Meaning:** The domain name specified in the URL could not be resolved by DNS.
  * **Resolution:** Check the spelling of the Server URL in your connection settings.
* **“SSL/TLS error”**
  * **Meaning:** The secure connection handshake failed.
  * **Resolution:** Check the certificate of your Tryton server. Is it expired? Does the certificate hostname match the configured URL? Self-signed certificates without a trusted root authority are rejected for security reasons.

## Analyzing Container Logs

When debugging runtime issues, inspect the logs generated by the Docker containers. Navigate to the install directory (`/opt/jaanos`) and run:

* **Inspect all container logs:**
  ```bash
  docker compose logs
  ```
* **Follow logs in real-time (Follow):**
  ```bash
  docker compose logs -f
  ```
* **Inspect application logs (`jaanos-suite`):**
  ```bash
  docker compose logs jaanos-suite
  ```
* **Inspect database logs (`db`):**
  ```bash
  docker compose logs db
  ```
* **Inspect proxy logs (`caddy`):**
  ```bash
  docker compose logs caddy
  ```
  *(Crucial for troubleshooting Let's Encrypt certificate acquisition issues).*
